## Conventions

- **Server Components by default.** A file gets `"use client"` only if it uses state, effects, or browser APIs.
- **One section per file** under `components/sections/`. Sections accept their copy as props — no fetching inside.
- **Data flow:** `.sitecraft/*.json` → `lib/sitecraft.ts` typed loader → page → section props. The agent never reads JSON directly inside a component.
- **Routes match `customer.json#pages[]` slugs.** If the dashboard adds a page later, the agent re-runs and adds the route — no manual edits.
- **No barrel files** (`index.ts` re-exports) except inside `components/ui/`.
